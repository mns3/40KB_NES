# 40KB_NES

## 🎮 프로젝트 소개
**40KB_NES**는 닌텐도 엔터테인먼트 시스템(NES/패미컴) 하드웨어 환경에서 구동되도록 제작된 게임
제한적인 8-bit 하드웨어 자원 안에서 효율을 추구하며, 게임 로직 구현하는데 중점을 둠

## 🛠 기술적 핵심 성과 (Technical Highlights)
본 프로젝트는 **하드웨어의 제약을 기술적으로 극복하고 리소스를 최적화**하는데 주력하였음.

### 1. 그래픽 데이터 파이프라인 및 최적화
* **바이너리 타일 데이터 직접 제어:**
    * 그래픽 에셋을 엔진에서 자동으로 불러오는 방식이 아닌, NES PPU(Picture Processing Unit)가 인식 가능한 **바이너리 타일 데이터(Binary Tile Data)로 직접 변환**하여 CHR ROM에 저장하고 관리하는 파이프라인을 구축함.
* **메모리 효율 극대화:**
    * 하드웨어 스프라이트 개수 제한과 용량 한계를 극복하기 위해 **미러링(Mirroring)** 기법과 **팔레트 스와핑(Palette Swapping)** 기술을 적극 활용함.
    * 동일한 타일 데이터를 반전시키거나 색상 인덱스만 교체하여 재사용함으로써 그래픽 메모리(VRAM) 사용량을 획기적으로 줄임.

### 2. 하드웨어 한계 극복을 위한 입력 시스템
* **Context-Sensitive Action 구현:**
    * A, B 두 개의 버튼만 존재하는 NES 컨트롤러의 물리적 한계를 극복하기 위해, 플레이어의 상태와 주변 환경(Context)에 따라 액션이 분기되는 시스템을 설계함.
    * 버튼 하나로 점프, 공격, 상호작용 등 다양한 액션을 상황에 맞게 수행하도록 하여 조작 편의성과 게임플레이의 깊이를 동시에 확보함.
